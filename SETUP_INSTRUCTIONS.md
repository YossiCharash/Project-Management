# הוראות התקנה - Setup Instructions

## התקנה ראשונית (Initial Setup)

### 1. הפעלת השרת
```bash
cd backend
python main.py
```

### 2. רישום מנהל ראשוני
עכשיו **אין** יצירה אוטומטית של מנהל דוגמה. אתה צריך ליצור את המנהל הראשוני בעצמך:

1. **גש ל**: `http://localhost:8080/admin-register`
2. **מלא את הפרטים שלך**:
   - שם מלא
   - כתובת אימייל
   - סיסמה חזקה
3. **לחץ על "צור מנהל"**
4. **תתחבר אוטומטית למערכת**

### 3. יצירת מנהלים נוספים
לאחר שהתחברת כמנהל ראשוני:

1. **גש לעמוד ניהול הזמנות**: `http://localhost:8080/admin-invites`
2. **לחץ על "צור הזמנה"**
3. **מלא פרטי המנהל החדש**
4. **קבל קוד הזמנה**
5. **שלח את הקוד למנהל החדש**

### 4. רישום מנהל חדש
המנהל החדש:

1. **גש ל**: `http://localhost:8080/admin-invite`
2. **הזן את קוד ההזמנה**
3. **מלא את הפרטים שלו**
4. **צור סיסמה**
5. **תתחבר אוטומטית**

## יתרונות המערכת החדשה

### ✅ אבטחה מלאה
- רק המנהל הראשי יכול ליצור מנהלים חדשים
- קודי הזמנה ייחודיים ופגי תוקף
- מניעת יצירת מנהלים לא מורשים

### ✅ ניהול קל
- ממשק משתמש פשוט ויעיל
- מעקב מלא אחרי הזמנות
- העתקה מהירה של קודי הזמנה

### ✅ גמישות
- תאריכי תפוגה מותאמים
- מחיקת הזמנות לא נוצלו
- מעקב אחרי כל הפעולות

## קישורים חשובים

- **התחברות**: `http://localhost:8080/login`
- **רישום מנהל ראשוני**: `http://localhost:8080/admin-register`
- **רישום עם קוד הזמנה**: `http://localhost:8080/admin-invite`
- **ניהול הזמנות**: `http://localhost:8080/admin-invites`
- **ניהול משתמשים**: `http://localhost:8080/users`

## הערות חשובות

1. **רק התקנה ראשונית**: רישום מנהל ישיר עובד רק אם אין מנהלים במערכת
2. **אבטחה**: כל הבדיקות מתבצעות בצד השרת
3. **גיבוי**: שמור את פרטי המנהל הראשוני במקום בטוח
4. **תחזוקה**: המערכת תומכת במספר מנהלים במקביל

## פתרון בעיות

### אם שכחת את פרטי המנהל הראשוני:
1. מחק את מסד הנתונים (אם זה בסביבת פיתוח)
2. הפעל מחדש את השרת
3. הירשם מחדש כמנהל ראשוני

### אם יש בעיה עם קודי הזמנה:
1. בדוק שהקוד לא פג תוקף
2. בדוק שהקוד לא נוצל כבר
3. צור קוד הזמנה חדש

המערכת מוכנה לשימוש! 🎉
